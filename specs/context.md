## Code Conventions

- **Naming:** snake_case for functions/variables, PascalCase for types/structs/enums, SCREAMING_SNAKE for consts
- **Error Handling:** `thiserror` for library error types, `eyre` for application layer; concrete error types with `?` propagation
- **State Management:** Lock-free approaches preferred: `scc` for concurrent maps/sets, `arc-swap` for read-heavy shared state, `parking_lot` for synchronous locks, `tokio::sync::Mutex` for async locks
- **Import Style:** Standard Rust conventions; types grouped by import source
- **TLS:** BoringSSL default, OpenSSL and Rustls as alternatives
- **HTTP:** `hpx` client with Tower middleware stack; forbidden: `reqwest`
- **Parsing:** `winnow` or `pest` preferred over ad-hoc manual parsing
- **Concurrent Maps:** `scc` over `dashmap` and `RwLock<HashMap<...>>`
- **Forbidden deps:** `anyhow` (use `eyre`), `log` (use `tracing`), `reqwest`, `dashmap`

## Architectural Decisions (Existing)

- Tower middleware stack for request/response processing
- Feature flag architecture with fine-grained features and presets (`hft`, `stealth`)
- Two WebSocket backends (yawc and fastwebsockets) selectable at compile time
- TLS backend abstracted (BoringSSL default, OpenSSL, Rustls alternatives)
- Browser emulation profiles injected via `hpx-emulation` trait
- Download engine uses state machine for segment lifecycle
- CDP client uses async message routing
- Connection pool manages state transitions
