## Project Overview

- **Language(s):** Rust 2024 edition, workspace with 6 library crates + 1 binary crate
- **Framework(s):** Tokio async runtime, Tower middleware, Axum (in yawc)
- **Package Manager:** Cargo (workspace resolver v3), justfile for task running
- **Architecture:** Monorepo workspace — `crates/` for libraries, `bin/` for CLI binaries

## Code Conventions

- **Naming:** snake_case for functions/variables, PascalCase for types, SCREAMING_CASE for constants
- **Error Handling:** `thiserror` for library error types, `eyre` for application layer
- **State Management:** Lock-free preferred (`scc`, `arc-swap`), `parking_lot` for sync locks, `tokio::sync::Mutex` for async locks
- **Import Style:** Standard Rust — stdlib, external crates, local crates, separated by blank lines
- **Linting:** Clippy pedantic + nursery enabled, `unwrap_used = "deny"`, `expect_used = "deny"`, `panic = "deny"`, `dbg_macro = "deny"`, `print_stdout = "deny"`, `print_stderr = "deny"`, `allow_attributes = "deny"`
- **Formatting:** Nightly rustfmt, taplo for TOML, rumdl for markdown

## Existing Test Patterns

- **BDD Tests:** `crates/hpx-dl/tests/cucumber.rs` with Gherkin features under `crates/hpx-dl/features/`
- **Property Tests:** `proptest` dependency present; usage varies by crate
- **Benchmarks:** `criterion` dependency present; `crates/yawc/` has benchmarks

## Architecture Decisions (Existing)

- AGENTS.md mandates: `thiserror` for libraries, `eyre` for apps, `scc` over `dashmap`, `hpx` over `reqwest`, `winnow`/`pest` over manual parsing
- Feature flag architecture: fine-grained features (`json`, `ws`, `boring`, etc.) with presets (`hft`, `stealth`)
- Tower middleware stack for request/response processing
- BoringSSL as default TLS backend with Rustls as alternative
- Two WebSocket backends (yawc and fastwebsockets) selectable at compile time
