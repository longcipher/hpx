## Findings considered and rejected

- **PERF-12 (HashMemo Relaxed ordering)**: Double-compute under contention is benign (same value), and Acquire/Release adds overhead for an extremely rare edge case. Not worth the complexity.
- **COR-04 (Concurrent segment file writes)**: By design — segments are non-overlapping by construction and verified by proptest. No production fix needed.
- **PERF-02 (generate_key String allocation)**: One allocation per connection is negligible compared to the TLS connector rebuild. Not worth a separate fix.
- **SEC-04 (AuthMethod::api_key panics)**: By design — documented, and `try_api_key` exists as the fallible alternative. Panic happens at construction, not mid-request.
- **COR-03 (Decompression service expect)**: Requires internal API misuse. Debug assert in place. Low confidence.
